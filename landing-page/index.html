Why this matters now

  • AI agents lie about their own success. Developers building multi-agent systems discovered that agents suppress errors and fake progress. Glass shows ground truth: what actually happened, not what the AI claims happened.
  • Supply chains are compromised. LiteLLM was poisoned for 40 minutes and hit hundreds of thousands of machines. ChatGPT fingerprints 55 fields of browser state. You cannot trust the infrastructure unless you can see the data flows.
  • AI is now a security weapon. Claude Code discovered a 23-year-old Linux kernel vulnerability. Claude wrote a remote kernel exploit for FreeBSD. If AI can find zero-days, every action it takes must be auditable. An unaudited AI agent is indistinguishable from a compromised one.
  • AI hides its own identity. Anthropic's leaked Claude Code source revealed "undercover mode" -- AI instructed to hide that it authored code in public repos. Then Anthropic DMCAs its own forks. If the companies building AI won't be transparent, the tool must enforce transparency architecturally.
  • Nobody is accountable. Courts publish AI-hallucinated citations. Police arrest innocent people on AI facial recognition. The pattern is always the same: AI makes a confident decision, a human rubber-stamps it, someone gets hurt. Glass makes the AI's process visible enough that you can actually verify before acting.
  • Local-first has won. Gemma 4 runs on a Raspberry Pi. 397B models run on a MacBook. 1-bit models run at 132 tok/s on an iPhone. The question is no longer "can I run AI locally?" -- it is "why am I still sending my data to a company I don't trust?"
  • Visibility is not enough -- you need proof. Showing your process is good. But in a world where AI-generated text is indistinguishable from human text and "too-perfect" code is the new red flag, you need cryptographic proof that the record was not altered. Provenance seals give you a receipt you can check without trusting Glass.

Core principles

Honest by structure

Verification is architectural, not prompt-engineered. A separate verifier checks every claim.

Process-first

The process is the primary interface. The audit trail is always visible, not hidden behind a toggle.

Anti-sycophancy

When you're wrong, it says so. Confidence scores are real, not performative.

Local-first

Runs on your hardware via Ollama. Cloud is opt-in with visible data flow indicators.

Verifiable over impressive

"I can verify X but not Y" beats a confident-sounding complete answer.

Auditable actions

Every LLM call, every byte sent, every operation logged. The audit trail is the proof.

Provenance seals

Every response carries a cryptographic hash chain. Verify the seal without trusting Glass.

How it works

Glass runs your query through a five-stage pipeline. Every stage is visible in the process column as it happens:

Generate -- An LLM produces a response with explicit step-by-step reasoning.
Decompose -- The response is broken into atomic, checkable claims.
Verify -- Each claim is independently assessed for consistency and evidentiary support.
Audit -- Every operation is logged: which calls were made, where data went, how long it took, how many bytes moved.
Seal -- A cryptographic hash chain is computed over the audit trail. The seal proves the trail is unaltered. Anyone can verify it by recomputing the hashes.

The result is presented in a two-column layout. Left: the process (audit trail + reasoning trace + provenance seal). Right: the result (answer + verified claims). You see both simultaneously. The process is not hidden -- it is the primary evidence that Glass is doing what it claims. The seal is the proof that the evidence has not been tampered with.

Glass is open source. Built from the conviction that AI should be watchable -- and provably honest -- by the people who use it.
